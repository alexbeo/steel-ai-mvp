## 3. Сценарий A — обучение модели (обязательный первый шаг)

Если моделей в `models/` нет, нужно сначала обучить хотя бы одну.

### Шаг 3.1. Вкладка «🤖 Обучение модели»

1. **Выберите класс стали** в верхнем dropdown:
   - `Pipe HSLA (API 5L X60-X70)` — для трубных сталей (ЦТЗ), target = предел текучести σт.
   - `Q&T Carbon Steels (EN 10083-2)` — углеродистые сталей европейского стандарта (C22/C35/C45/C60), target = твёрдость HRC.

2. **Target property** — dropdown автоматически отфильтрован под выбранный класс. Выберите что прогнозировать:
   - HSLA: σт / σв / δ / KCV-60.
   - Q&T: HRC / σв.

3. **Optuna trials** — ползунок 10-150. Рекомендация:
   - **10-20** для быстрого демо (~30-60 сек обучение).
   - **40-50** для нормального качества (~1-2 мин).
   - **100+** для продакшена (~3-5 мин).

4. Нажмите **«🤖 Обучить модель»**. Прогресс-бар заполняется.

### Шаг 3.2. Отчёт после обучения

```
✅ Модель en10083qt_hardness_hrc_xgb_20260423_204613 готова

R² test     MAE test     R² train     Coverage 90% CI
0.982       1.46         0.996        77.2%

⚠️ Отчёт Critic
[HIGH] M02: Calibration нарушена: 90% CI покрывает 77.2% точек
            (ожидалось 85-95%). Overconfident uncertainty.
💡 Применить conformal prediction на hold-out для калибровки.

🤖 LLM-Critic (Claude Sonnet 4.6)   ← только если ANTHROPIC_API_KEY задан
(observations 0-5 штук или caption «проблем не обнаружено»)

Feature importance
(bar chart top-15 фичей)
```

**Как читать:**
- **R² test > 0.7** — модель работоспособна. < 0.5 → не доверять.
- **MAE test** — средняя абсолютная ошибка прогноза. Для σт ≈ 10-15 МПа считается хорошо.
- **Coverage 90% CI 85-95%** — доверительный интервал откалиброван. Вне диапазона — Critic поднимает warning, но модель не блокируется.
- **Pattern Library warnings** (⚠️) — автоматические проверки на ML-патологии (overfit, data leakage, physics-mismatch). Читайте, но HIGH-warning ≠ остановка обучения в демо-UI.
- **LLM-Critic** — второе мнение от Claude. Появляется только при наличии API-ключа.

## 4. Сценарий B — прогноз по конкретному составу

Инженер-технолог вводит состав → получает прогноз с uncertainty.

### Шаг 4.1. Вкладка «📊 Прогноз»

1. В sidebar выберите обученную модель (класс покажется badge'ом).
2. Поля ввода подстроятся под feature_set активной модели:
   - HSLA → 16 полей (химия C/Mn/Nb/Ti/... + rolling_finish_temp + cooling_rate + n_ppm).
   - Q&T → 10 полей (химия C/Mn/Si/P/S/Cr + austenitizing_temp + tempering_temp + tempering_time + section_thickness).
3. Заполните значения. Физические границы уже вшиты (min/max), вылезти за них нельзя.
4. Нажмите **«🔮 Предсказать»**.

### Шаг 4.2. Результат прогноза

```
Твёрдость, HRC: 47.3 ± 1.8
90% ДИ: [45.5, 49.1]

⚠️ Состав вне training distribution — прогноз ненадёжен!   ← если OOD
```

Для HSLA дополнительно показываются производные параметры: CEV(IIW), Pcm, CEN, микролегирование. Используются Critic-ом для проверки свариваемости.

**Как читать:**
- **Точечный прогноз ± полуширина CI** — сколько реально можно ждать.
- **OOD flag** (красный alert) — состав вне обучающего распределения, прогноз экстраполяционный. Опытную плавку не делать без перепроверки.

## 5. Сценарий C — Inverse design со стоимостью (только HSLA)

Металлург хочет: «Найди **самый дешёвый** состав под σт 485-580 МПа с CEV ≤ 0.43».

### Шаг 5.1. Подготовка

Выберите в sidebar **HSLA-модель**. Если активна Q&T модель, вкладка «🎯 Дизайн» покажет баннер «inverse design пока HSLA-only».

### Шаг 5.2. Вкладка «🎯 Дизайн сплава»

1. **Целевые свойства**: введите σт min и max (например, 485-580 МПа).
2. **Ограничения**: CEV(IIW) max (типично 0.43 для трубной стали), Pcm max (0.22).
3. **Дополнительные параметры NSGA-II** (expander) — population_size и n_generations. По умолчанию 80/60 — нормально.

### Шаг 5.3. Expander «💰 Прайс материалов» (развёрнут по умолчанию)

...

**Что можно делать:**

## 6. Сценарий D — Al-раскисление на ковше (on-line advisory)

Металлург-плавильщик перед выпуском из печи получил замер активного кислорода O_a — нужно добавить Al для раскисления. 

### Шаг 6.1. Вкладка «🔥 Раскисление»

Шапка:
```
Активный класс: pipe_hsla   Target O_a из профиля: 5 ppm
Термодинамическая модель: [Fruehan 1985 — Fruehan R., Ladle Metallurgy, ISS 1985 ▼]
```

Переключатель модели содержит 3 варианта:
- **Fruehan 1985** (дефолт, классика) — точность ±40 ppm.
- **Sigworth-Elliott 1974 (JSPS)** — ±30 ppm.
- **Hayashi-Yamamoto 2013** — ±20 ppm при [Al] > 0.05%.

### Шаг 6.2. Forward — «Сколько Al нужно»

Подвкладка по умолчанию. Вводите замеры с пробы:

```
O_a измерено, ppm:   450    T расплава, °C:  1620
Масса стали, т:      180    Целевой O_a:      5    ← prefilled из профиля
% активного Al:      100    Угар, %:         20
Heat ID (опц.):  (например: H-20260423-017)
```

Нажмите **«🧮 Рассчитать»**.

### Шаг 6.3. Результат Forward

```
💊 Навеска Al: 112.5 кг (0.625 кг/т)
 ├─ Активный Al на реакцию: 90.0 кг
 ├─ Угар: 22.5 кг (20%)
 └─ Ожидаемый остаточный O_a: 5.0 ppm (цель)

💰 Стоимость: 270 EUR (при Fruehan 1985)

[💾 Сохранить в Decision Log]
```

**Что делать с результатом:**
- Распечатать или записать для плавильщика — указать навеску Al и Heat ID в журнале.
- «💾 Сохранить» (опционально) — пишет запись в Decision Log с тегом `deoxidation` для audit-trail.

### Шаг 6.4. Inverse — «Качество Al по факту»

Плавка прошла, есть замеры **до** и **после** присадки. Проверяем эффективное качество Al-лигатуры:

```
O_a до, ppm:        500    O_a после, ppm:   10
Al добавлено, кг:    65    T, °C:           1620
Масса стали, т:     180    Угар, %:           20
```

Нажмите **«🔍 Оценить качество»**.

Результат:
```
Эффективное активное Al: 82.4 %

- Реально сработал (связал O): 53.5 кг
- Ожидался при 100% чистоте: 52.0 кг
- Допущение burn_off: 20%
```

Если < 70% — появляется предупреждение «подозрительная поставка».

### Шаг 6.5. Compare-mode — сравнение моделей

Подвкладка **«⚖️ Сравнить модели»** — те же входы, но запускаются все 3 формулы:

```
Модель                Al, кг   Al, кг/т   O_a, ppm   Цена, EUR
Fruehan 1985          112.5    0.625      5.0        270.0
Sigworth-Elliott 1974 112.5    0.625      5.0        270.0
Hayashi-Yamamoto 2013 112.5    0.625      5.0        270.0

Разброс между моделями: ±0.0% — ожидаемая неопределённость ...
```

Bar chart снизу наглядно показывает разницу. Полезно для разговора с металлургом-скептиком: «вот три академические формулы, их результаты различаются на N% — это неопределённость физики, не нашего метода».

## 7. Сценарий E — аудит-trail во вкладке «📚 История»

Все решения pipeline'а, каждый сохранённый расчёт раскисления, каждое обучение модели попадают в Decision Log (SQLite).

### Шаг 7.1. Вкладка «📚 История»

Фильтр по фазе: `training / inverse_design / deoxidation / reporting / meta`. Slider «Максимум записей» задаёт limit.

Каждая запись раскрывается в expander:
```
[training] Обучена модель en10083qt_hardness_hrc_xgb_... — 2026-04-23 (model_trainer)
  Reasoning: XGBoost с time-based split + GroupKFold по campaign_id. Optuna HP tuning (40 trials). Quantile regression для uncertainty. Test R²=0.982, MAE=1.46, coverage 90% CI=77.2%.
  Альтернативы: CatBoost, MLP, Random Forest
  Context: {...всё в json}
  Теги: xgboost, hardness_hrc, production_candidate
```

### Шаг 7.2. Типичные случаи использования

- Через 3 месяца: «почему в июле мы взяли состав X с дорогим Nb?» → фильтр `inverse_design` → видно snapshot цен, ограничения, топ-кандидатов.
- Аудит: «какие модели обучали за квартал?» → фильтр `training` → список с версиями, R², MAE.
- Для LLM-Critic: фильтр tag=`llm_critic` (через API) — показывает сколько токенов потратил Claude, cache_read (эффективность caching), реплики модели.

## 8. Частые вопросы

### Модель показывает R² test = 0.2 — почему?
- Обычно: Optuna trials слишком мало. Попробуйте 40+.
- Или: неправильный target для класса (например, задали HRC для HSLA-генератора — там этой колонки нет).
- Проверьте warnings «⚠️ Отчёт Critic» — часто там и причина.

### Inverse design выдаёт пустой Pareto front
- Constraints слишком жёсткие (например, σт 650-700 при CEV ≤ 0.40). Ослабьте.
- Increase population_size / n_generations.
- Проверьте OOD flags на кандидатах — если все оранжевые, модель обучена на других диапазонах.

### Раскисление: получаю «Навеска Al = 0 кг»
- Target O_a задан >= измеренного O_a (DX02 warning). Перепроверьте значения проб или ТЗ.

### Раскисление: эффективная чистота < 50%
- Либо поставка действительно плохая (проверить лабораторию).
- Либо реальный угар на ковше выше заложенных 20% (попробуйте 30-35% в Inverse-форме).
- Либо O-анализатор нуждается в калибровке (проверьте с Celox-пробой).

### Как остановить Streamlit
```bash
lsof -ti:8501 | xargs kill
```

- **Модели**: `models/<version>/` — XGBoost artifacts + meta.json (gitignored).
